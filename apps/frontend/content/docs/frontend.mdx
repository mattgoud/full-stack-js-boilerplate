## 2. Directory Structure

```txt
apps/frontend/
├── app/                 # App Router Pages & Layouts
│   ├── globals.css      # Global CSS & Tailwind v4 Config
│   └── layout.tsx       # Root Layout
├── content/docs/        # MDX Documentation files
├── components/          # React Components
│   ├── ui/              # Shadcn/UI Primitives
│   └── ...              # Feature-specific components
├── lib/
│   ├── source.ts        # Fumadocs configuration
│   └── utils.ts         # Utilities (cn, formatters)
└── public/              # Static Assets
```

### Rendering Strategy

<Callout title="Server Components First" type="info">
  We adhere to the **"Server First"** mentality of [Next.js 16](https://nextjs.org/docs/app/building-your-application/rendering/server-components).
</Callout>

- **Server Components (Default)**: Used for data fetching, layouts, and non-interactive UI.
- **Client Components ("use client")**: Used strictly for event listeners, stateful hooks, and browser APIs.
- **Static Site Generation (SSG)**: Documentation pages are statically generated using [Fumadocs MDX](https://fumadocs.dev/docs/mdx).

### Tailwind CSS v4

<Callout title="Modern Configuration" type="warn">
  Tailwind v4 removes the need for `tailwind.config.js`. All configuration (theme, colors, spacing) is handled directly in CSS using the `@theme` directive in `apps/frontend/app/globals.css`.
</Callout>

---

<Callout title="Performance Tip" type="idea">
  Always prefer CSS variables over hardcoded colors to ensure full compatibility with the [Theme Switcher](/dashboard).
</Callout>

## 4. Monorepo Integration

The frontend **never** duplicates types defined in the backend. It imports them from the shared package.

```typescript
import { UserDto } from "@repo/shared";
```
