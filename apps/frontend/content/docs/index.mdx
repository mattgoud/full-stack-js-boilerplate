## 1. Architecture

The project is organized as a monorepo using **Turborepo** and **pnpm** workspaces. Documentation is independent and accessible at `/docs` using **Fumadocs**.

## 2. AI Agent & Living Documentation

This boilerplate is designed to be **AI-First**. It includes a "Living Documentation" workflow that ensures code and documentation stay in sync.

- **Agent Context**: Optimized for **Gemini CLI** via the `.gemini/` folder.
- **Rules & Skills**: Custom instructions guide AI agents through complex tasks.
- **Sync Verification**: Use the `doc-check` tool to ensure architectural changes are documented.

## 3. Getting Started

### Prerequisites

- **Node.js**: Latest LTS.
- **pnpm**: `v10.28.1`.
- **Docker**: Required for database.

### Installation

```bash
pnpm install
```

### Running the Project

Choose one of the following methods to run the application:

#### Option 1: Local Development (Recommended)
1. **Start the database**: `docker compose up db -d`
2. **Run dev servers**: `pnpm dev`

#### Option 2: Full Docker
Run the entire stack inside containers:
```bash
docker compose up
```

## 4. Key Commands

| Command | Description |
| :--- | :--- |
| `pnpm dev` | Start all apps in development mode. |
| `pnpm build` | Build all packages and applications. |
| `pnpm lint` | Run ESLint across the monorepo. |
| `pnpm turbo check-types` | Run global TypeScript type checking. |
| `pnpm doc-check` | Verify if documentation is in sync with code changes. |
| `pnpm --filter backend prisma:migrate` | Run database migrations. |
