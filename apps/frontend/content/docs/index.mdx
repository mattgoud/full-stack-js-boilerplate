## 1. Architecture

<Callout title="Modern Tech Stack" type="info">
  This boilerplate uses the latest versions of Next.js, NestJS, and Prisma, optimized for performance and type safety.
</Callout>

The project is organized as a monorepo using [Turborepo](https://turbo.build/) and [pnpm](https://pnpm.io/) workspaces. Documentation is independent and accessible at `/docs` using [Fumadocs](https://fumadocs.dev/).

<Cards>
  <Card title="Frontend" href="/docs/frontend" icon={<LayoutIcon />}>
    Next.js 16+, React 19, Tailwind CSS v4, Shadcn/UI.
  </Card>
  <Card title="Backend" href="/docs/backend" icon={<ServerIcon />}>
    NestJS 11 REST API, Prisma ORM.
  </Card>
  <Card title="Shared" href="/docs/backend#4-monorepo-integration" icon={<BoxIcon />}>
    Shared TypeScript types, DTOs, and utilities in @repo/shared.
  </Card>
  <Card title="Infrastructure" href="/docs/backend#3-development" icon={<DatabaseIcon />}>
    Docker Compose for PostgreSQL and services.
  </Card>
  <Card title="Authentication" href="/docs/frontend#6-authentication" icon={<LockIcon />}>
    GitHub OAuth, JWT session management, and server-side route protection.
  </Card>
</Cards>

## 2. AI Agent & Living Documentation

This boilerplate is designed to be **AI-First**. It includes a [Living Documentation](/docs/ai-guide) workflow that ensures code and documentation stay in sync.

- **Agent Context**: Optimized for [Gemini CLI](https://github.com/google-gemini/gemini-cli) via the `.gemini/` folder.
- **Rules & Skills**: Custom instructions guide AI agents through complex tasks.
- **Sync Verification**: Use the `doc-check` tool to ensure architectural changes are documented.

## 3. Getting Started

<Accordions>
  <Accordion title="Prerequisites">
    Ensure you have the following installed:
    - [Node.js](https://nodejs.org/) (Latest LTS)
    - [pnpm](https://pnpm.io/) `v10.28.1`
    - [Docker](https://www.docker.com/) (for database)
  </Accordion>

  <Accordion title="Installation">
    Clone the repository and install dependencies:

    ```bash
    pnpm install
    ```
  </Accordion>

  <Accordion title="Running the Project">
    <Tabs items={['Local Development', 'Full Docker']}>
      <Tab value="Local Development">
        Start the database: 
        ```bash
        docker compose up db -d
        ```
        Run dev servers: 
        ```bash
        pnpm dev
        ```
      </Tab>
      <Tab value="Full Docker">
        Run the entire stack inside containers:
        ```bash
        docker compose up
        ```
      </Tab>
    </Tabs>
  </Accordion>
</Accordions>

## 4. Key Commands

| Command | Description |
| :--- | :--- |
| `pnpm dev` | Start all apps in development mode. |
| `pnpm build` | Build all packages and applications. |
| `pnpm test` | Run all tests across the monorepo. |
| `pnpm lint` | Run [ESLint](https://eslint.org/) across the monorepo. |
| `pnpm check-types` | Run global TypeScript type checking. |
| `pnpm doc-check` | Verify if documentation is in sync with code changes. |
| `pnpm --filter backend prisma:migrate` | Run [Prisma](https://www.prisma.io/) migrations. |
| `pnpm --filter backend prisma:seed` | Seed the database with initial data. |

## 5. Development Conventions

  Mandatory strict mode for TypeScript.
